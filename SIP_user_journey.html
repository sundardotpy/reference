SIP User Journey

SIP, in bonds allows user to invest a fixed amount every month in bonds selected by Wint Wealth. The process is automated, similar to mutual fund SIPs. It helps investors build a disciplined habit of investing regularly and also supports diversification over time. In simple terms, it works like a regular bond investment. The only difference is that the payment happens automatically on the selected date. All other aspects of the bond, such as issuer, coupon, tenure, repayment, and risk, remain the same.

Journey overview

End-to-end SIP flow

The SIP order behaves like a regular bond order after creation, but the payment is collected through the active Autopay mandate for recurring deductions.

1

Select SIP theme

User selects the SIP theme, date of monthly deduction, and investment amount.

High Rated and High Returns generally use amount multiples of ₹10,000.

2

Review issuer pool

User sees issuers available under the selected theme pool.

The user can modify the pool by excluding issuers they do not want.

3

Pay first installment

The first installment is paid on the same day as a regular order through UPI or net banking.

The initial order still gets the SIP benefit where applicable.

4

Set up Autopay

After initial payment, the user sets up the Autopay mandate for recurring deductions.

UPI is used for lower ticket SIPs. eNACH is used for higher ticket SIPs.

5

Monthly notification

User gets a notification 5 days before deduction with the bond chosen for that month.

The SIP order can be skipped only until T-2 days.

6

Deduction & settlement

Autopay deduction happens on the chosen date. Order settles on the next working day.

Once settled, the bond appears like a normal bond in the portfolio.

First order timing

Initial payment and recurring debit logic

The first installment is paid immediately when SIP is set up. The recurring Autopay deduction depends on whether there is a minimum 10-day gap from setup to the chosen deduction date.

Chosen date is 1-22

If the user sets up SIP on 12 April and chooses any date from 1 to 22, the first Autopay deduction will happen in May.

Initial order: 12 Apr Recurring debit: May 1-22

Chosen date is 23-30

If the user sets up SIP on 12 April and chooses a date from 23 to 30, the recurring deduction can happen in April itself.

Initial order: 12 Apr Recurring debit: Apr 23-30
Mandate setup

UPI mandate vs eNACH mandate

The mandate is the NPCI/payment instruction. SIP is Wint Wealth’s internal recurring investment configuration. Both need to be understood separately.

Parameter UPI mandate eNACH mandate
Typical use case Useful for ₹10,000 SIP setup. Required for ₹20,000 and above SIP setup.
Mandate limit Amount per cycle can be up to ₹15,000. This is the maximum cap, not always the exact debit amount. Can be set for higher limits. Modification is supported up to ₹2,90,000.
Monthly debit amount Actual debit depends on bond price on deduction date. Actual debit depends on bond price on deduction date.
Modification Amount cannot be modified. Date can be modified. Both amount and deduction date can be modified.
Support timeline Modification requests usually take 1-2 working days. Modification requests usually take 1-2 working days.
Product rules

Key aspects

Monthly amount may not be exactly flat

If the user selects ₹10,000 SIP, the actual monthly debit may be around ₹10,021 or ₹9,989.

This happens because bond prices fluctuate daily due to accrued interest and fair price movement. The selected amount is the intended SIP size. The actual debit depends on the bond price on the deduction date.

Active SIP order blocks cancellation

SIP cannot be cancelled when an active SIP order exists. The user must skip the order within the allowed window or wait until the order is settled.

Skip till T-2 T-1 not cancellable Wait for settlement

Cancellation & failure model

Common edge cases

User asks to cancel on T-1

Cancellation is not possible after the NPCI pre-debit stage. The user needs to wait for debit and settlement, then cancellation can be handled if no active SIP order remains.

Waiting stateNot cancellable

Insufficient funds

If funds are not available on debit day, the SIP order may fail and the SIP may get terminated. Wint Wealth does not charge a fine, but the bank may levy charges as per its policy.

Failure riskBank policy applies

User cancels mandate directly

If the user cancels the mandate before NPCI notification or before debit, it can cancel the mandate and the linked SIP flow. No deduction should happen in this hard-cancel scenario.
