Support handling

Use these only after verifying the current stage. Avoid confirming completion unless the final deletion stage is visible or confirmed internally.

Use with verification
When deletion request is raised

Sir/Madam, I can see that your account deletion request has been raised. The next step is the Account Closure Form e-sign, which has been sent to registered email. Please complete the e-sign through the link shared with you so the closure flow can proceed.

note: At the request-raised stage, no direct support action is required as per the notifier reference.
When ACF link or e-sign is involved

Sir/Madam, the Account Closure Form is required for completing the closure flow. Since the link is shared over email when the required checks are in place, please complete the e-sign from the same link. Once the form is signed, the closure process will move to the next stage.

Expectation setting: Do not say the account is closed only because the ACF is signed. ACF signing is an intermediate stage before final deletion.
When account deletion is completed

Sir/Madam, your account closure has been initiated. As part of the closure flow, the account will be deleted along with the Demat account with in 5-7 working days.

Use this only after final confirmation: This script should be used only when the account-deleted stage is verified.
Checklist

Before responding to the customer

A clean checklist reduces incorrect promises and helps the team set the right expectation.

Ready reference
✓
Check the latest Slack notifier stage
Identify whether the case is at request raised, ACF signed, or final deletion.
Mandatory
✓
Separate customer action from internal action
Do not ask the customer to do anything where the stage clearly says user action is NA.
Best practice
✓
Confirm ACF status before closure expectation
If the ACF is not signed, guide the user to complete the e-sign from the shared link.
Dependency
✓
Use final closure language only after confirmation
Mention account and Demat account deletion only when the final stage is confirmed.
Avoid error

Do

  • Verify the latest stage before responding.
  • Use simple language with clear next step.
  • Explain that ACF signing is part of the closure flow.
  • Set closure expectation only after final confirmation.

Don’t

  • Do not say the account is deleted at the request-raised stage.
  • Do not ask for customer action when user action is marked NA.
  • Do not skip KYC / ACF dependency while explaining the flow.
  • Do not use final closure script without verification.
